# 项目完成标准 (Project Done Definition)

本文件用于明确项目"完成"的标准，避免无限完善循环。

## 核心原则

**项目已处于"功能完成"状态**。后续修改必须满足以下条件之一：
1. 修复导致编译错误的 BUG
2. 修复导致运行时崩溃的 BUG
3. 修复导致核心玩法无法进行的 BUG
4. 用户明确要求的具体功能

## 禁止的"完善"行为

以下行为属于**过度完善**，除非用户明确要求，否则不要执行：
- 添加新的音效、VFX、动画
- 调整数值平衡（除非用户要求）
- 添加新的成就、皮肤、装饰
- 重构已经能正常工作的代码
- 添加注释、文档（除非用户要求）
- 添加边缘情况处理（除非会导致崩溃）

## 如果用户说"继续"

当用户说"继续"时，**不要**主动寻找新的完善点。应该：
1. 询问用户具体想做什么
2. 或者检查是否有未完成的明确任务
3. 或者报告项目已完成

## 修改流程

任何修改必须遵循：
1. **明确问题**：用户指出的具体问题
2. **最小修改**：只修改解决问题所需的代码
3. **验证**：编译通过即可，不要过度测试
4. **停止**：修复完成后停止，不要继续找其他问题
