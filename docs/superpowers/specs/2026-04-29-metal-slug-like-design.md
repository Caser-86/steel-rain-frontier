# 《钢雨前线》游戏设计文档与技术栈

日期：2026-04-29

## 1. 项目概述

《钢雨前线》是一款 2D 横版跑轰射击游戏，核心参考《合金弹头》的爽快节奏、夸张爆炸、密集敌群、载具关卡和 Boss 战。项目重点不是复刻老式像素，而是做“现代精品 2D 跑轰”：角色优美、动作细腻、场景层次丰富、敌人和地形变化多、武器形态切换有策略。

目标体验：

- 玩家 5 秒内明白操作，30 秒内进入战斗高潮。
- 每 2-3 分钟出现新敌人、地形机制、武器或事件。
- 第一关长度约 22-28 分钟，像一个完整小电影，但用检查点切成清晰段落。
- 画面追求高质量手绘角色、精细爆炸特效、强光影层次和流畅动画。

核心卖点：

- 多角色切换，每名角色性格、机动性、技能和武器偏好不同。
- 多武器形态切换，同一把武器可变成不同战术功能。
- 大型长关卡，包含海滩、村落、雨林、遗迹、铁轨、堡垒。
- 多样敌人与地形，避免单调站桩射击。
- 高表现力 2D 美术：骨骼动画、像素特效、2D 动态光、天气和破坏效果。

## 2. 游戏类型与平台

类型：

- 2D 横版动作射击
- 单人优先
- 后续支持本地双人合作

目标平台：

- 第一阶段：Windows PC
- 第二阶段：Steam Deck / 手柄优化
- 第三阶段：Switch / PlayStation / Xbox 视预算决定

目标性能：

- PC：1920x1080，60 FPS
- Steam Deck：1280x800，60 FPS
- 场景中同屏敌人 15-25 个，子弹 80-150 发，爆炸特效可控降级

## 3. 世界观与美术风格

### 3.1 世界设定

近未来边境战争。人类在热带群岛发现古代能源遗迹，军阀、企业军和变异生物同时涌入。玩家小队“钢雨”执行登陆突袭，目标是摧毁敌方堡垒并夺回遗迹核心。

整体气质：

- 战争冒险
- 高燃但不严肃到沉重
- 角色鲜明，有轻微漫画感
- 场景美丽但危险

### 3.2 画面方向

推荐风格：高清手绘角色 + 精致像素特效 + 2D 光影。

角色：

- 角色使用 2D 骨骼动画，保证动作优美流畅。
- 面部、发丝、服装挂件、武器后坐力都要有细节。
- 每名角色有独特站姿、跑姿、换弹动作、受击动作。

场景：

- 多层视差背景。
- 前景草叶、灰尘、雨水、火星增加深度。
- 使用 2D 灯光突出爆炸、枪口火焰、遗迹能量。

特效：

- 爆炸用像素风或手绘序列帧。
- 枪口火焰、烟雾、碎片、弹壳都要明显。
- 大 Boss 受击时有装甲破裂、火花、黑烟。

## 4. 核心循环

单局循环：

1. 进入关卡。
2. 清理敌群。
3. 获得武器/弹药/救援奖励。
4. 通过地形挑战。
5. 遭遇小 Boss 或事件战。
6. 解锁新路线或检查点。
7. 进入 Boss 战。
8. 结算表现，获得角色经验、武器熟练度、收藏品。

长期循环：

- 解锁角色皮肤、武器模组、角色剧情档案。
- 重玩关卡挑战隐藏路线和高评分。
- 后续开放高难度、Boss Rush、双人合作。

## 5. 操作设计

基础操作：

- 移动：左右移动
- 跳跃：短按小跳，长按高跳
- 下蹲/趴伏：降低受击面积，可射击低角度目标
- 翻滚/闪避：短无敌帧，穿过部分攻击
- 射击：支持八方向瞄准
- 近战：贴身自动或按键触发
- 切换武器形态：快速循环当前武器形态
- 切换角色：关卡检查点或特定补给站切换，后期可做实时小队切换
- 使用技能：角色专属技能，有冷却

手感原则：

- 跳跃响应立即生效。
- 射击不能让角色过度僵硬。
- 翻滚要有明确启动、无敌、收尾窗口。
- 受击后短暂无敌，避免连续硬直。
- 武器后坐力有表现，但不破坏移动爽感。

## 6. 可选角色

### 6.1 艾拉 Aila

定位：均衡队长。

个性：冷静、可靠、带一点讽刺幽默。

能力：

- 标准生命值和移动速度。
- 翻滚无敌帧略长。
- 换弹速度快。
- 主动技能：战术标记。标记范围内敌人受伤提高，弱点高亮。

适合玩家：

- 新手
- 想稳定通关的人

### 6.2 赫维 Harvey

定位：重装火力。

个性：豪爽、话多、喜欢正面推进。

能力：

- 生命值高，移动慢。
- 正面受伤降低。
- 可短时间架起防爆盾，挡住小型子弹。
- 重型武器后坐力低。

主动技能：压制火线。持续扫射前方区域，击退普通敌人。

适合玩家：

- 喜欢站稳输出
- 喜欢火力压制

### 6.3 燕 Yan

定位：高速突击。

个性：张扬、挑衅、爱冒险。

能力：

- 移动最快。
- 可空中冲刺。
- 近战伤害高。
- 生命值低，容错少。

主动技能：影步斩。瞬移到近距离敌人身后造成高伤害。

适合玩家：

- 高操作玩家
- 喜欢近战和空中机动

### 6.4 林熙 Lin Xi

定位：工程支援。

个性：理性、毒舌、专注机械。

能力：

- 可部署小型炮塔。
- 可放感应地雷。
- 修复载具和补给箱概率提高。
- 对机械敌人有额外伤害。

主动技能：无人机蜂群。短时间召唤 3 架小无人机自动攻击。

适合玩家：

- 喜欢布置和控场
- 喜欢探索隐藏物

### 6.5 诺亚 Noah

定位：狙击/能量爆发。

个性：沉默、精准、神秘。

能力：

- 基础生命值最低。
- 蓄力射击可穿透多个敌人。
- 命中弱点时返还部分技能能量。
- 远距离伤害高。

主动技能：裂隙光束。发射短时高伤害直线激光。

适合玩家：

- 喜欢高风险高爆发
- 喜欢精确射击

## 7. 武器系统

### 7.1 武器设计原则

每把武器都有 3 种形态。切换形态改变攻击节奏、弹道、范围或附加效果。玩家不只是“捡更强武器”，而是根据敌人和地形选择形态。

通用规则：

- 武器有弹药。
- 默认手枪无限弹药。
- 强武器弹药有限。
- 形态切换无消耗，但部分形态耗弹更多。
- 角色对某些武器有熟练加成。

### 7.2 武器列表

#### 突击步枪

- 连射形态：稳定输出，适合普通敌人。
- 穿甲形态：射速低，穿透盾牌和薄装甲。
- 榴弹形态：消耗高，抛物线爆炸。

#### 霰弹枪

- 散射形态：近距离大范围。
- 独头弹形态：远距离单发高伤害。
- 燃烧形态：留下短时火焰区域。

#### 重机枪

- 压制形态：高射速，移动变慢。
- 旋转形态：预热后伤害升高。
- 破甲形态：低射速高硬直伤害。

#### 火焰喷射器

- 持续火焰：近距离持续伤害。
- 火球：中距离爆裂。
- 地面燃烧：封锁地形，适合防守。

#### 能量枪

- 激光形态：直线持续伤害。
- 电链形态：弹跳多个敌人。
- 护盾脉冲：短距离击退子弹和敌人。

#### 火箭筒

- 直射火箭：高伤害。
- 分裂火箭：命中后散出小爆炸。
- 追踪火箭：锁定空中单位，伤害较低。

## 8. 敌人设计

### 8.1 敌人类型

普通兵：

- 步枪兵：基础射击。
- 冲锋兵：快速接近，逼迫玩家移动。
- 手雷兵：抛物线攻击，压迫掩体。

防御兵：

- 盾兵：正面挡子弹，需要绕后、穿甲或爆炸。
- 重甲兵：行动慢，硬直高。

远程兵：

- 狙击手：出现红色瞄准线，延迟开火。
- 迫击炮兵：打玩家位置，需要移动躲避。

特殊兵：

- 喷火兵：近距离危险，死亡后可能爆炸。
- 医疗兵：给敌人回血，优先击杀目标。
- 通讯兵：呼叫增援。

机械单位：

- 侦察无人机：飞行，骚扰。
- 机枪炮塔：固定火力点。
- 四足机甲：小 Boss 级敌人。
- 装甲车：移动平台和火力压制。

变异体：

- 爬行者：贴地移动，可从墙面出现。
- 爆囊怪：靠近自爆。
- 巨臂兽：近战砸地，制造冲击波。

### 8.2 AI 原则

- 敌人组合制造战术问题，而不是堆数量。
- 普通兵负责节奏，特殊兵改变优先级，机械单位制造火力区域，变异体改变移动方式。
- 敌人必须有明显预警动画。
- 玩家死亡应觉得“我可以躲”，不是被暗算。

## 9. 地形与关卡机制

地形类别：

- 滩头：炮击坑、浅水、登陆艇残骸。
- 村落：可破坏木屋、屋顶路线、人质房间。
- 雨林：藤蔓、泥地、伏击草丛、毒虫巢。
- 壕沟：低矮掩体、机关枪火线、手雷压制。
- 遗迹：升降石台、能量门、碎裂地板。
- 铁轨：高速载具段、岔路、移动障碍。
- 堡垒：电梯、闸门、炮台、警报增援。

互动机制：

- 可破坏掩体。
- 可救援 NPC。
- 隐藏补给箱。
- 可爆油桶。
- 陷阱触发器。
- 多路线分支。
- 载具拾取和损毁。

## 10. 第一关设计：《黎明登陆线》

目标长度：22-28 分钟。

关卡主题：从海滩登陆一路推进到敌方堡垒，展示游戏全部基础能力。

### 10.1 关卡结构

#### 段落 1：潮汐滩头

时长：3-4 分钟。

内容：

- 教玩家移动、跳跃、射击、趴伏。
- 敌人从登陆艇残骸和沙袋后出现。
- 远处炮击制造视觉冲击。
- 第一个补给箱给突击步枪。

重点：

- 开场必须快，30 秒内开枪。
- 爆炸、海浪、火焰营造第一印象。

#### 段落 2：悬崖村落

时长：4-5 分钟。

内容：

- 玩家进入敌占村落。
- 木屋可破坏，屋顶可走。
- 救 3 名人质，分别奖励弹药、生命、隐藏钥匙。
- 引入盾兵和手雷兵。

重点：

- 展示地形立体路线。
- 给玩家第一次选择：正面推进或屋顶绕行。

#### 段落 3：雨林壕沟

时长：5 分钟。

内容：

- 敌人从草丛伏击。
- 泥地降低移动速度。
- 炮击区需要快速穿过。
- 小 Boss：四足侦察机甲。

小 Boss 机制：

- 阶段 1：机枪扫射，玩家躲进壕沟。
- 阶段 2：跳跃砸地，制造冲击波。
- 阶段 3：暴露背部核心，鼓励绕后。

#### 段落 4：古代遗迹

时长：4-5 分钟。

内容：

- 场景由战争转向神秘遗迹。
- 引入变异体。
- 能量机关改变平台位置。
- 遗迹光源与雨林阴影形成画面高潮。

重点：

- 展示画面优美和风格差异。
- 用怪物改变敌人节奏。

#### 段落 5：铁轨补给站

时长：4-5 分钟。

内容：

- 玩家抢夺装甲摩托或轻型装甲车。
- 自动横向滚动，玩家射击前后敌人。
- 路线分上下两条。
- 追逐敌方装甲列车。

重点：

- 提供速度变化。
- 减少步行疲劳。

#### 段落 6：堡垒大门 Boss

时长：5-6 分钟。

Boss：巨门战争机“巴别-01”。

阶段 1：

- 双机炮横扫。
- 导弹雨标记地面。
- 玩家破坏左右炮台。

阶段 2：

- 巨门打开，露出核心反应炉。
- 敌兵从两侧增援。
- Boss 使用激光扫射。

阶段 3：

- 核心过载。
- 平台坍塌。
- 玩家在移动残骸上输出最终核心。

结尾：

- 堡垒爆炸。
- 遗迹深处亮起未知信号。
- 解锁第二关：地下实验室。

### 10.2 检查点

检查点设置：

- 滩头结束
- 村落中段
- 村落结束
- 壕沟小 Boss 前
- 壕沟小 Boss 后
- 遗迹结束
- 载具段前
- Boss 前
- Boss 阶段 3 前

死亡惩罚：

- 回到最近检查点。
- 保留剧情进度。
- 当前强武器弹药按检查点状态恢复，不无限刷。

### 10.3 隐藏内容

- 村落屋顶隐藏霰弹枪。
- 雨林藤蔓后隐藏能量枪。
- 遗迹符文门需要救满 3 名人质开启。
- 铁轨上路线可获得角色档案碎片。
- Boss 前隐藏补给可恢复载具生命。

## 11. Boss 设计原则

Boss 必须满足：

- 至少 3 个阶段。
- 每阶段有明确新攻击。
- 每个攻击都有预警。
- 有弱点窗口。
- 场景参与 Boss 战。
- 视觉破坏随血量变化。

第一关 Boss 目的：

- 测试玩家移动、跳跃、趴伏、形态切换、清杂兵。
- 给玩家“打穿战争机器”的爽感。
- 不做过高难度，保证第一关通关率。

## 12. UI 与反馈

战斗 UI：

- 左上：角色头像、生命、技能冷却。
- 右上：当前武器、当前形态、弹药。
- 下方小提示：人质、隐藏物、当前检查点。
- Boss 战：Boss 血条分阶段显示。

反馈原则：

- 捡武器有短暂停顿和大图标提示。
- 切换形态用图标和音效确认。
- 受击方向有屏幕边缘提示。
- 弱点命中有特殊火花和音效。
- 弹药不足提示必须明显，但不能遮挡战斗。

## 13. 音乐与音效

音乐风格：

- 军鼓、工业电子、管弦打击、民族乐器点缀。
- 第一关从紧张登陆音乐逐渐转向遗迹神秘音乐，Boss 时进入强节奏战斗曲。

动态音乐：

- 平时探索层。
- 敌人出现加入鼓点。
- 小 Boss 加入低音和铜管。
- 大 Boss 进入完整战斗主题。

音效重点：

- 每把武器声音必须区分明显。
- 爆炸有低频冲击。
- 弹壳、换弹、齿轮、警报强化机械感。
- 角色有短语音，但不频繁打扰。

## 14. 技术栈

### 14.1 推荐主栈

引擎：

- Unity 6.3 LTS

语言：

- C#

渲染：

- URP 2D Renderer
- 2D Lights
- Shader Graph

动画：

- Spine 4.x 或 Unity 2D Animation
- Timeline 用于过场和 Boss 入场

地图：

- Unity Tilemap 为主
- Tiled 作为外部关卡草图工具可选

输入：

- Unity Input System
- 键鼠和手柄同时支持

音频：

- FMOD for Unity 2.03

资源管理：

- Addressables
- Git LFS 管理大图、音频、动画文件

构建：

- Unity Build Profiles
- GitHub Actions 或本地构建脚本后续再加

测试：

- Unity Test Framework
- PlayMode 测试核心战斗逻辑
- EditMode 测试数据配置

### 14.2 为什么选 Unity

Unity 适合本项目，因为：

- 2D 工具链成熟。
- URP 2D 光照适合做精美横版画面。
- Spine、FMOD、Aseprite、Tiled 等生态支持好。
- PC 和主机发布路径更成熟。
- 团队招聘和外包资源更容易。

不首选 Godot 的原因：

- Godot 2D 很强，开源免费。
- 但商业素材管线、主机发布、外包协作生态弱于 Unity。
- 如果预算极低且只做 PC，可重新评估 Godot。

不首选 Unreal 的原因：

- 2D 项目成本偏高。
- Paper2D 生态弱。
- 对本项目不是最省力选择。

## 15. 工程架构

推荐模块：

- Core：游戏状态、时间、事件总线、对象池。
- Player：角色控制、角色能力、受击、技能。
- Weapons：武器数据、形态、弹道、弹药。
- Enemies：AI、感知、攻击模式、精英词缀。
- Levels：关卡触发器、检查点、地形机关。
- Bosses：Boss 状态机、阶段脚本。
- UI：HUD、菜单、提示、结算。
- Audio：FMOD 事件封装、动态音乐控制。
- VFX：特效池、屏幕震动、命中特效。
- Save：存档、解锁、设置。

核心数据方式：

- 使用 ScriptableObject 配置角色、武器、敌人、关卡事件。
- 逻辑代码不硬编码数值。
- 关卡触发器通过事件调用刷怪、镜头、音乐、检查点。

## 19. 风险与控制

风险 1：美术成本过高。

控制：

- 先做 1 个角色精美完整，其余角色做简化可玩版。
- 先定动画骨架规范。
- 特效复用材质和粒子模块。

风险 2：第一关太长导致疲劳。

控制：

- 每 3-4 分钟一个新变化。
- 插入载具段和遗迹段改变节奏。
- 检查点密集，失败不重复太多。

风险 3：武器形态太多导致平衡困难。

控制：

- MVP 只做 3 把武器。
- 每种形态只负责一个明确用途。
- 数据驱动调参。

风险 4：敌人太多导致 AI 混乱。

控制：

- 敌人行为组件化。
- 先做基础兵、盾兵、飞行单位、装甲单位四大原型。
- 后续通过参数和攻击模式变体扩展。

风险 5：视觉很美但可读性差。

控制：

- 敌人子弹使用固定高亮色。
- 角色轮廓保持清楚。
- 爆炸和背景亮度不能盖住敌方攻击。
- Boss 预警统一颜色和音效。

## 20. MVP 验收清单

必须完成：

- 角色移动、跳跃、趴伏、翻滚手感稳定。
- 八方向射击可用。
- 3 把武器、9 种形态可切换。

- 8-10 分钟关卡。
- 检查点、死亡重生、弹药补给。
- 基础 HUD。
- 基础音效。
- 60 FPS。

暂不做：

- 在线联机。
- 复杂角色养成。

- 大量剧情分支。
- 多结局。

## 21. 下一步建议

下一步不是直接做完整游戏，而是做垂直切片。

推荐顺序：

1. 建 Unity 项目。
2. 做角色控制器。
3. 做武器形态系统。
4. 做敌人基础 AI。
5. 做第一关前 3 分钟灰盒。
6. 放入 1 名精美角色和 1 套高质量场景样张。
7. 调手感和节奏。
8. 扩展到 8-10 分钟垂直切片。

核心判断标准：

- 如果移动和射击不好玩，先别堆内容。
- 如果第一名角色不够美，先别画 5 名角色。
- 如果一种武器三形态不好用，先别扩 6 把武器。
